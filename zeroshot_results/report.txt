  WHAT busyBeaver DOES

================================================================================

1. Prompt engineering — Expert coder framing + markdown code output
2. pass@3 multi-temperature retry — Pushes ~80% → 89% on HumanEval
3. Code extraction — Parses markdown ```python blocks cleanly
4. Sandboxed test execution — 15s timeout, crash recovery, no infinite loops
5. Checkpointing — Resume from any crash point (critical for 500-problem runs)

The model writes the code. The harness measures it fairly.

================================================================================

  HONEST CAVEATS

================================================================================

1. pass@3 vs pass@1: We use 3 attempts per problem. Command A+'s published
   scores are likely pass@1. However, our pass@1 for HumanEval would be ~80%,
   still above Command A+'s 75%.

2. Model choice: Qwen2.5-Coder-3B is purpose-built for code. This isn't a
   random small model — it's one of the best code-specialized models at this
   size. The point is that specialization beats general-purpose scale on
   narrow tasks.

3. MMLU-Pro: Fundamental architecture mismatch. A code-specialized model
   cannot memorize the breadth of world knowledge that a 218B general model
   can. This is expected and not a harness limitation.

4. MBPP parity: The 3B model is roughly on par with Command A+ on the harder
   code benchmark (MBPP problems have more complex specs than HumanEval).
